## The "Side-Database" Approach
The key to making this work seamlessly is a .NET concept called a **Side-Database (`Database.ReadDwgFile`)**.

Your add-in does not need to open the standard template file in the user's AutoCAD editor. Instead, your C# code can open the template file in the computer's memory silently, extract all the correct layers and properties, and compare them against the active drawing.

### The Logic Workflow
1. **Load Standard:** The plug-in silently opens your standard template DWG in memory.
2. **Build Dictionary:** It reads the `LayerTable` of the standard DWG and builds an internal dictionary mapping layer names to their correct colors, linetypes, line weights, and descriptions.
3. **Analyze Active DWG:** It scans the active drawing's `LayerTable`.
4. **Compare & Correct:**
   * **If a layer matches the standard name:** It updates the color, linetype, etc., to match the standard DWG.
   * **If a layer is non-standard but close (e.g., via a mapping rule):** It renames it or merges the entities over.
   * **If a new standard layer is missing:** It clones the layer from the standard DWG using `WblockCloneObjects`.

## 2. Implementing AutoCAD Navigation in WPF

To get that smooth middle-mouse pan and scroll-wheel zoom inside your WPF node canvas, you will need to wrap your canvas in a transformation matrix and override the mouse event handlers.

In WPF, you use a `MatrixTransform` applied to your main Node Canvas, and then listen for three specific mouse events:

### The C# Mouse Navigation Logic

```csharp
// Conceptual WPF Code-Behind for the Node Canvas
private Point _panStart;
private Matrix _currentMatrix = Matrix.Identity;

// 1. ZOOM: Zooming with the Scroll Wheel
private void NodeCanvas_MouseWheel(object sender, MouseWheelEventArgs e)
{
    var position = e.GetPosition(NodeCanvas);
    double scaleFactor = e.Delta > 0 ? 1.1 : 1.0 / 1.1;

    // Zoom centered on the mouse cursor position (Just like AutoCAD!)
    _currentMatrix.ScaleAt(scaleFactor, scaleFactor, position.X, position.Y);
    CanvasTransformer.Matrix = _currentMatrix;
}

// 2. PAN START: Detect Middle Mouse Button Press
private void NodeCanvas_MouseDown(object sender, MouseButtonEventArgs e)
{
    if (e.ChangedButton == MouseButton.Middle)
    {
        _panStart = e.GetPosition(this);
        NodeCanvas.CaptureMouse();
    }
}

// 3. PAN MOVING: Move canvas relative to mouse movement
private void NodeCanvas_MouseMove(object sender, MouseEventArgs e)
{
    if (NodeCanvas.IsMouseCaptured && e.MiddleButton == MouseButtonState.Pressed)
    {
        var currentPos = e.GetPosition(this);
        Vector delta = currentPos - _panStart;
        _panStart = currentPos;

        _currentMatrix.Translate(delta.X, delta.Y);
        CanvasTransformer.Matrix = _currentMatrix;
    }
}
```

## 3. Optimizing the "Tall Graph" Performance

While AutoCAD users are completely fine navigating a massive visual space, WPF can get sluggish if it tries to render 2,000 visual nodes simultaneously. This is called visual tree bloating.

To keep the UI running at a locked 60 FPS while panning:
* **UI Virtualization:** Only actually render the nodes that are currently visible within the user's screen boundary (the viewport). As the user pans down, nodes entering the screen are drawn, and nodes leaving the screen are temporarily unloaded.
* **Simplified Node Geometry:** Keep the node shapes clean. Avoid heavy drop-shadow effects or complex gradients on the nodes themselves. Use simple borders and clean text blocks. Let the colored bezier curves (the connection lines) be the main visual focus.
